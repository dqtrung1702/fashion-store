Mô hình dữ liệu Fashion Store

Dữ liệu nghiệp vụ chính nằm trong MongoDB. File ảnh nằm trong MinIO; MongoDB chỉ lưu đường dẫn ảnh hoặc tên object để sản phẩm tham chiếu. Redis được dùng làm cache mềm cho dữ liệu public đọc nhiều.

1. Tổng quan collection

Collection Vai trò Chỉ mục chính
users Lưu tài khoản khách hàng và quản trị viên. email unique, username unique.
products Lưu catalog sản phẩm, biến thể, giá, tồn kho, ảnh, nhãn hiển thị và thông tin SEO. slug unique, text index cho name/description, index theo status, collectionSlug, collectionSlugs, category, categories, price.
catalog_collections Lưu danh mục/collection dùng ở storefront và màn hình admin. slug unique, index theo isActive/sortPriority.
site_content Lưu một gói nội dung CMS cho header/footer, trang chủ, landing page, editorial page và trang thông tin. key unique, hiện dùng giá trị site-content.
carts Lưu giỏ hàng của người dùng đã đăng nhập. user_id unique, updated_at.
wishlists Lưu danh sách yêu thích của người dùng đã đăng nhập. user_id unique.
orders Lưu đơn hàng, bản chụp thông tin sản phẩm tại thời điểm mua, trạng thái và lịch sử theo dõi. user_id, email, created_at.

5. Gói nội dung website

Cấu trúc chính

  • siteChrome: nội dung thương hiệu, announcement, footer, thời lượng giữ hàng và ảnh nền website.
  • homePageContent: nội dung trang chủ.
  • merchandisingPages: các trang bán hàng/landing page theo chiến dịch.
  • editorialPages: nội dung bài viết hoặc lookbook.
  • infoPages: trang thông tin như shipping, returns, privacy.

Cách lưu

  • Lưu thành một document trong collection site_content.
  • Khóa document hiện tại là site-content.
  • Admin gọi POST /api/admin/content/import để upsert bundle này.
  • Nội dung hỗ trợ locales.vi/locales.en; backgroundImage được đồng bộ như trường dùng chung cho cả hai ngôn ngữ.

6. Giỏ hàng, yêu thích và đơn hàng

Nhóm dữ liệu Trường quan trọng Ghi chú nghiệp vụ
Giỏ hàng user_id, items, subtotal_amount, shipping_amount, total_amount Item lưu product_id, variant_id/sku và snapshot tên/ảnh/giá. Khi trả về API, dịch vụ API serialize lại với catalog mới nhất nếu còn tìm thấy sản phẩm.
Yêu thích user_id, items Item lưu product_id và added_at. API trả về danh sách sản phẩm đã serialize để frontend hiển thị.
Đơn hàng orderNumber, user_id, email, items, các trường tổng tiền, status, shipping_address, contact_phone, trackingSteps Item trong đơn hàng là snapshot tại thời điểm mua. Sau khi tạo đơn, dịch vụ API trừ tồn kho theo variant.

7. Trạng thái đơn hàng

Giá trị Ý nghĩa
pendingĐơn mới tạo, chờ xác nhận.
confirmedĐơn đã được xác nhận.
processingĐơn đang được chuẩn bị.
shippedĐơn đã giao cho đơn vị vận chuyển.
deliveredĐơn đã giao thành công.
cancelledĐơn đã hủy.
Phí vận chuyển hiện được tính theo quy tắc trong dịch vụ API: miễn phí nếu subtotal lớn hơn 200, ngược lại phí là 18.

8. Tài sản ảnh trong MinIO

Trường Ý nghĩa
object_nameTên object trong MinIO. API kho ảnh sản phẩm chỉ làm việc với prefix products/....
urlURL public được tạo từ MINIO_PUBLIC_SECURE, MINIO_PUBLIC_ENDPOINT, bucket và object name.
content_typeLoại file được lấy từ upload content type hoặc suy ra từ phần mở rộng file.
sizeDung lượng file theo byte.
last_modifiedThời điểm object được cập nhật lần cuối trong MinIO.
product_usage_countSố sản phẩm đang tham chiếu ảnh này trong coverImage hoặc images.

9. Quy tắc xóa ảnh

  • Giao diện admin hiển thị số sản phẩm đang dùng từng ảnh trong thư viện MinIO.
  • Nếu product_usage_count lớn hơn 0, nút xóa bị khóa ở màn hình admin.
  • Dịch vụ API vẫn kiểm tra lại trước khi xóa. Nếu ảnh đang được sản phẩm dùng, API trả lỗi 409 để tránh mất ảnh sản phẩm.
  • Chỉ ảnh thuộc prefix products/ được liệt kê và xóa qua API quản trị kho ảnh sản phẩm.

10. Redis cache

Redis không phải nguồn dữ liệu gốc. Nếu Redis lỗi, API bỏ qua cache và đọc MongoDB trực tiếp.

Nhóm cache Key/pattern TTL Invalidation
Danh sách/chi tiết sản phẩmcache:products:*120 giây, search 60 giâyXóa khi admin tạo/sửa/xóa sản phẩm hoặc import catalog.
Danh sách/chi tiết collectioncache:collections:*600 giâyXóa khi admin tạo/sửa/xóa collection hoặc import catalog.
Danh sách categorycache:categories:*600 giâyXóa cùng catalog cache.
Content publiccache:content:*600 giâyXóa khi admin import/lưu content.

11. Excel import/export sản phẩm

  • Excel được xử lý ở frontend bằng thư viện xlsx; backend vẫn nhận payload sản phẩm/collection qua API admin hiện có.
  • Admin có thể tải template Excel, điền nhiều dòng sản phẩm và import hàng loạt. Ảnh có thể để trống rồi bổ sung trong admin sau.
  • Export Excel hiện xuất dữ liệu sản phẩm hiện tại và bỏ qua phần ảnh theo yêu cầu vận hành.
  • Các cột danh mục dùng slug collection; nhiều danh mục có thể nhập dạng danh sách ngăn cách bằng dấu phẩy hoặc xuống dòng tùy template.

12. Dữ liệu phía trình duyệt

Một số state vẫn có lớp dữ liệu dự phòng local trong giao diện để tránh làm hỏng trải nghiệm khi API chưa sẵn sàng hoặc người dùng chưa đăng nhập.

Nhóm dữ liệu Cách dùng hiện tại
Giỏ hàng kháchGiỏ hàng của khách chưa đăng nhập lưu ở trình duyệt.
Yêu thích kháchDanh sách yêu thích của khách chưa đăng nhập lưu ở trình duyệt.
Dữ liệu catalog/nội dung dự phòngStore giao diện còn có dữ liệu dự phòng local và hàm reset ở tầng state, nhưng các nút reset nguy hiểm đã được gỡ khỏi màn hình admin.
